First of all we have to make some changes in the dataset. Upon checking the dataset, you can see that some prices have -1 value which means the price for that record is missing. Also some records have NaN values for some features which means the value for that feature is missing. There are multiple ways to approach these records. We can simply drop them since we don't have info about them, although this method can be implemented without any problem, sometimes it'll perform badly and also it'll result in smaller dataset if the missing values are a lot which is always an undesired phenomena. Another way of approaching missing values is using mean or median of all the values for that feature instead of the missing value. This method is good for small dataset and surely saves us from losing data which really important but it won't perform well in large datasets and also it can only be used for numerical values. Another way of approaching missing values is using the most frequent value of that feature instead of missing value or simply putting a new unique value for missing data. This method will solve the problem of the last method since it can be used for categorical data but it can't be used for numerical values and also addition of new features to the model while encoding may result in poor performance.

For this project I've used all the methods above since we have NaN values both in categorical(brand) and numerical(mileage and year) form. By experimenting I came to the conclusion that removing missing data will result in lower RMSE so I used this method. For price feature since it'll be our result there's no other way than removing them so whatever method you use you have to remove prices that have -1 value. Also since mileage can be any value it made more sense to me to use mean for it but for year since it must be a natural number it made more sense to me to use median for it.

Also there were some values in year equal to <1366 which I replaced them with 1365 to have a numerical value instead.

Some info + Question 2¶

In machine learning we want numerical values but most of the time there will be categorical values in our dataset. We can convert these values to numerical values. Like here in our dataset brand and category are categorical data. We can deal with categorical values in two different ways. The first one is using label encoding, which assigns one number to each category. The second one is using one-hot encoding, which will assign different columns to each category. Before we start transforming our data, it is important to figure out if the feature we are working on is ordinal or nominal. An ordinal feature is best described as a feature with natural, ordered categories and the distances between the categories is unknown. We can use LabelEncoder for ordinal features not nominal features. In nominal features we cant replace the categories by a number since this would imply the features have an order, which is wrong for brand and category in our dataset. So we should use a nominal encoding. The most popular way to encode nominal features is one-hot-encoding. Essentially, each categorical feature with n categories is transformed into n binary features. If the number of the categorical features is low (relative to the amount of data) one-hot encoding will work best. But if the number of the categorical features is large and the dataset is small, one-hot encoding may not be feasible, and regressor may not be able to efficiently learn.

I have tested both of the methods and to be honest both of them had quite the same accuracy at the end but according to the reason stated above for using one hot encoding and since we have 26 brands and our data is large enough I prefered to use one hot encoding. And also since one hot encoding values are 0 or 1 it can't get biased like in label encoding if the number of unique values are a lot, one hot is kinda normalized and the range of its values is much less.

Some Info + Question 3¶

After implementing the procedure of tdidfVectorizer myself and the implementation taking too much time (near 10 minutes for each run) I decided to use tfidfVectorizer from sklearn which made everything much more easier. We'll give the vectorizer the sentences from last part and by choosing max_features and setting it to desired value like n we'll get tf-idf of top n words. I also used word names as column identifiers to know what each tf-idf refers to.

My implementation is commented here. It counts the number of word occurrence in each sentence and also records the number of sentences having a particular word it also records the number of occurrence of each word so that we can choose the top words as our features. tf-idf is calculated by it's formula which is:

tf(t,d) = count of word t in sentence d / number of words in sentence d

df(t) = occurrence of word t in documents (sentences)

idf(t) = log(N/(df + 1)) N: Total number of sentences

$tf-idf(t, d) = tf(t, d) * log(N/(df + 1))$

I have used different values for max_feature 50, 100, 150, 250, 500 and in all of them there wasn't a huge different in RMSE so by try and error I used the one that gave the best possible RMSE.

def evaluateModel(test_actual , test_predict , train_actual , train_predict):

    evaluationDF.loc["MSE" , "Train"] = metrics.mean_squared_error(train_actual , train_predict) 
    
    evaluationDF.loc["RMSE" , "Test"] = np.sqrt(metrics.mean_squared_error(test_actual , test_predict))

Question 6¶

If we use a large max_depth value it'll make the model more complex meaning that there's higher chance of overfitting but if we use a small max_depth value it'll make the model simpler meaning that there's higher chance of underfitting and here's the age old tradeoff between simplicity and consistency we have to check which value generates the best result for our model. Also using a higher value for max_depth will make the training process take a long time but on the contrary smaller value will give us the result quickly.

max_depth is maximum depth of the tree if we don't assign a value to it then nodes are expanded until all leaves are pure or until all leaves contain less than min_samples_split samples. It's for knowing until what point we should continue to produce nodes.

min_samples_split is the minimum number of samples required to split an internal node. It's for determining the label at the end meaning that if the number of remaining samples in the node is less than this value we'll determine the label for that node and stop producing nodes.

Both of the mentioned parameters above were explained in the course lectures as well.

Lower min_samples_split will cause overfitting so I set it to a high value.

Hyperparameters of Random forest¶

max_depth is the maximum depth of the tree if the value is too high it can take a really long time to get the results and the model will be really complex also other issues like overfitting can happen.

n_estimators is the number of trees in the forest and get results from all of them and finally determine the result based on them. Raising this value will result in larger number of trees so we can create a more robust aggregate model with less variance at the cost of a greater training time.

max_features is the number of features to consider when looking for the best split meaning that it'll choose a portion of the features based on the set value. This was the game changer and decreased RMSE the most. A small value which means less features considered when splitting at each node will reduce the variance at the cost of higher individual tree bias. High value will obviously have inversed results.

Some info + Question 7¶

As the results show above we can see that random forest performs much better than decision tree and we were expecting such behaviour since random forest gets info from multiple trees and finally generates the result in comparision to decision tree which is only one tree.

Bias: Difference between the average prediction of our model and the correct value which we are trying to predict. Model with high bias pays very little attention to the training data and oversimplifies the model. It always leads to high error on training and test data. So when bias is high it'll mean that our model is simple so it'll result in underfitting

Variance: Variability of model prediction for a given data point or a value which tells us spread of our data. Model with high variance pays a lot of attention to training data and does not generalize on the data which it hasnt seen before. As a result, such models perform very well on training data but has high error rates on test data. So when variance is high it'll mean that model is too complex and also performs better on train set rather than test set so it'll result in overfitting

From bias point of view since random forest gets info from several trees there's small chance of underfitting so it'll be better and perform better. Form variance point of view decision trees are prone to overfitting especially when a tree is particularly deep but random forest rarely overfits so in this case is better too so it'll perform better. All in all since random forest gets information from multiple trees and basically consults and take a look at each of the trees' results before making a decision it'll have better results. One way Random Forests reduce variance is by training on different samples of the data. A second way is by using a random subset of features that we used here in the project as well. In random forest if we use many trees in our forest, eventually many or all of our features will have been included. This inclusion of many features will help limit our error due to bias and error due to variance. The achieved results clearly show that random forest performs better.
